# Reels Bot

Telegram-бот: загружаешь видео → Whisper распознаёт речь → выбираешь стиль/шрифт/позицию субтитров → FFmpeg рендерит красивое видео в стиле Reels.

## Возможности

- 🎙 Распознавание речи на любом языке (Groq Whisper, бесплатно)
- 🎨 **7 стилей**: Панч, Огонь, Караоке, Мята, Розовый, Неон, Классика
- 🔤 **8 трендовых шрифтов** с кириллицей: Montserrat, Russo One, Oswald, Unbounded, Rubik, Caveat (рукопись), Pattaya (кисть), Yeseva (элегант)
- 🌈 **Свой цвет** текста (11 вариантов)
- 📍 **Позиция текста**: сверху / по центру / снизу
- ✨ Анимация слов (эффект «выпрыгивания») + подсветка ключевых слов ИИ
- 👁 **Превью картинкой** — 1 кадр перед полным рендером
- 🎞 **Авто-вертикаль** — горизонтальное видео сам оборачивает в 9:16 с размытым фоном
- 🔊 Улучшение звука (нормализация громкости голоса)
- ✏️ Редактирование текста (замена слов и строк)
- 🎨 **Свои пресеты** — бот запоминает последний стиль/шрифт/цвет/позицию
- 💎 **PRO-подписка 99₽/мес** (безлимит): ЮKassa или ручная оплата с подтверждением
- 👥 **Рефералы** — пригласи друга, оба получают +видео
- 📊 Аналитика для админа (`/stats`)
- ⏳ Очередь рендеров (не тормозит при нагрузке)

## Требования

- Python 3.11+ (работает и на 3.14)
- **ffmpeg-full** (`brew install ffmpeg-full`) — нужен для субтитров
- Telegram Bot Token ([@BotFather](https://t.me/BotFather))
- Groq API Key (бесплатно, без карты — [console.groq.com](https://console.groq.com))

## FFmpeg (важно)

Обычный `brew install ffmpeg` **не умеет субтитры**. Нужен ffmpeg-full:

```bash
brew install ffmpeg-full
```

Бот сам находит ffmpeg-full, PATH настраивать не обязательно.

## Быстрый старт

```bash
cd reels-bot
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# впиши TELEGRAM_BOT_TOKEN и GROQ_API_KEY в .env
python -m bot.main
```

## Запуск в фоне (без терминала) 🚀

Чтобы бот работал сам, стартовал при включении Mac и перезапускался при сбоях:

```bash
bash service.sh install    # установить и запустить в фоне
bash service.sh status     # проверить статус
bash service.sh logs       # смотреть логи
bash service.sh stop       # остановить
bash service.sh restart    # перезапустить
```

После `install` терминал можно закрыть — бот продолжит работать.
Он также **сам перезапускается при изменении кода** (через watchfiles).

## Как пользоваться

1. Пришли боту видео с речью (до 20 MB; горизонтальное — сам сделает 9:16)
2. Бот распознает текст и покажет меню
3. Выбери **Стиль / Шрифт / Цвет / Позиция** кнопками
4. Жми «👁 Превью» — увидишь кадр без полного рендера
5. При желании — «✏️ Текст» (замена слов)
6. Жми «🎬 Сделать видео»

## Редактирование текста

- Заменить слово: `неправильное = правильное` (можно несколько строк)
- Одну строку целиком: `3 = новый текст`
- Или пришли весь текст заново (каждая фраза с новой строки)

## Команды

- `/start` — приветствие (поддерживает реф-ссылку `?start=<id>`)
- `/status` — остаток видео
- `/buy` — купить PRO (99₽/мес)
- `/grant <id> [дней]` — выдать PRO вручную (только админ)
- `/invite` — пригласить друга (+видео обоим)
- `/help` — помощь
- `/stats` — статистика (только админ)

## Переменные окружения

| Переменная | Описание |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен бота |
| `GROQ_API_KEY` | ключ Groq (бесплатный) |
| `FREE_VIDEO_LIMIT` | бесплатных видео на пробу (по умолч. 2) |
| `ADMIN_USER_ID` | твой Telegram ID для `/stats` и `/grant` (узнать: [@userinfobot](https://t.me/userinfobot)) |
| `PRICE_RUB` | цена PRO в рублях (по умолч. 99) |
| `SUB_DAYS` | длительность подписки в днях (по умолч. 30) |
| `REFERRAL_BONUS` | сколько видео за приглашение (по умолч. 3) |
| `MAX_CONCURRENT_RENDERS` | сколько видео рендерить одновременно (по умолч. 2) |
| `PAYMENT_PROVIDER_TOKEN` | токен ЮKassa из BotFather (пусто = ручная оплата) |
| `PAYMENT_INFO` | реквизиты для ручной оплаты (текст для пользователя) |
| `DB_PATH` | (опц.) путь к базе — для Railway-волюма |
| `FFMPEG_PATH` | (опц.) путь к ffmpeg вручную |

## Структура

```
reels-bot/
  bot/
    main.py                # точка входа
    handlers.py            # Telegram: меню, кнопки, сессии
    db.py                  # SQLite лимиты
    config.py              # настройки + Groq клиент
    services/
      transcribe.py        # Whisper + word-тайминги
      style_text.py        # стилизация текста (LLM)
      subtitles.py         # стили/шрифты/позиции → ASS
      render_video.py      # FFmpeg рендер
      ffmpeg_utils.py      # поиск ffmpeg/ffprobe, размеры
      pipeline.py          # подготовка + рендер
  assets/fonts/            # трендовые шрифты (кириллица)
  run.sh                   # запуск с авто-перезапуском
  service.sh               # фоновый автозапуск (launchd)
  data/                    # база + логи (создаётся)
```

## Монетизация (99₽ / мес)

Два режима, переключаются переменной `PAYMENT_PROVIDER_TOKEN`:

1. **Ручная оплата (по умолчанию, работает сразу, без ИП):**
   - Пользователь жмёт «💎 PRO», видит реквизиты из `PAYMENT_INFO` (СБП/карта).
   - Оплачивает, жмёт «✅ Я оплатил», присылает чек.
   - Тебе в личку падает заявка с кнопкой-подсказкой `/grant <id>`.
   - Ты проверяешь оплату и выдаёшь PRO: `/grant 12345` (или `/grant 12345 30`).

2. **Автооплата ЮKassa (когда оформишь самозанятость/ИП):**
   - В BotFather → Payments → подключаешь ЮKassa → получаешь provider token.
   - Вставляешь его в `PAYMENT_PROVIDER_TOKEN` — счёт в рублях выставляется автоматически,
     PRO включается сразу после оплаты. Ничего вручную подтверждать не нужно.

## Деплой на Railway (бот работает 24/7 без твоего Mac)

1. Залей папку `reels-bot` в GitHub-репозиторий (без `.env` — он в `.gitignore`).
2. На [railway.app](https://railway.app) → **New Project → Deploy from GitHub repo**.
3. Railway сам увидит `nixpacks.toml` (ставит ffmpeg) и запустит `python -m bot.main`.
4. В **Variables** добавь переменные из `.env` (TELEGRAM_BOT_TOKEN, GROQ_API_KEY, ADMIN_USER_ID, PRICE_RUB, PAYMENT_INFO и т.д.).
5. Чтобы база не сбрасывалась при редеплое: **Add Volume**, mount path `/app/data`,
   и переменную `DB_PATH=/app/data/bot.db`.
6. Готово — бот в сети постоянно. Логи и метрики видно в панели Railway.

> На Railway (Linux) обычный `ffmpeg` уже с поддержкой субтитров — `ffmpeg-full` нужен только на macOS.
